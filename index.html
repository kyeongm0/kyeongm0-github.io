설문조사 및 게시판 사이트

Spring Framework를 사용하여 진행한 설문조사 및 게시판 사이트입니다.

로그인 시 JWT를 사용하였습니다. JWT에 ID와 PW를 보내 로그인을 요청하면 Header, PayLoad, Signature를 정의하는데, 이 때 Payload에 Id나 회원등급 등의 정보를 담아서 게시물의 수정 권한이나 설문 답변의 통계자료 등 등급에 따라 접근에 제한을 두었습니다. 이 때 HTTP 요청을 처리하기 위해 axios를 사용하였습니다. axios로 받아온 데이터를 state에 새로 정의한 UserInfo에 값을 넣어서 필요할 때 마다 사용하였습니다.

모든 리스트 작업은 ditpatch를 이용해 바로 읽어올 수 있도록 하였고, 클릭 시 액션함수를 이용하여 GET 방식으로 현재 page 값을 넘겨줍니다. Spring boot는 MVC 구조를 따르기 때문에 Controller에서 page값을 받아서 page를 계산라는 함수를 실행시킵니다. User Domain에 pagination을 추가해 pagination의 정보를 User에 담아서 다시 보내서 state에 리스트와 page 정보를 담아줍니다.

모든 디테일 작업에서 등급에 따라 권한을 달리하기 위해서 v-show를 이용하였습니다. 회원은 Role을 USER만, 관리자는 USER와 ADMIN 둘 다 가지고 있기 때문에 ADMIN Role을 가지고 있는 경우만 수정에 접근하지 못하게 막을 수 있습니다. 따라서 관리자 등급은 회원이 작성한 게시물을 보거나 삭제는 가능하지만 수정은 할 수 없습니다.

반대로 설문지에 대한 통계자료나 회원 등급 변화는 관리자만 접근할 수 있습니다. Spring Security를 사용하여 Controller에 @PreAuthorize("hasRole('ROLE_USER') or hasRole('ROLE_ADMIN')") annotation을 추가하였습니다. 회원은 ADMIN Role을 가지고 있지 않기 때문에 통계자료에 접근할 수 없습니다.

게시물 작성 시 이미지 파일 업로드를 하기 위해 v-file-input을 사용하였으며 Spring boot에서 UUID를 이용해 랜덤한 업로드명을 부여하여 동일한 업로드명이 없도록 하였습니다. 동시에 파일 원본 이름도 같이 db에 저장하였고 resultMap과 collection을 이용하여 리스트화 하여 다중 파일 업로드도 가능하도록 하였습니다.

게시물 작성 시 group의 변수를 만들고 LAST_INSERT_ID를 이용하여 게시물의 Idx와 group의 값이 같게 하여 게시물에 대해 분류를 하였고, Order By를 사용하여 최신에 작성한 게시물이 가장 위로 오게 하였습니다. 또, order의 변수를 만들어 게시물에 대한 답글이 게시물 작성날짜를 기준으로 최신 게시물이 바로 아래에 오도록 하였으며 마지막으로 depth의 변수를 만들어 답글의 답글 또한 순서에 맞게 하였습니다.

회원 디테일 창에서 등급 변경 시 Admin Role을 부여하게 하여 @PreAuthorize("hasRole('ROLE_USER')") 뿐만 아니라 hasRole('ROLE_ADMIN')" 에도 접근 가능하도록 설계하였습니다.

게시물 디테일 창에서는 글에 대한 댓글을 쓸 수 있도록 하였으며 게시물과 마찬가지로 group, order, depth변수를 이용하여 최근의 댓글이 가장 위로 오도록 하였습니다. 댓글에 대댓글을 달 때와 댓글의 수정은 Components에 있는 vue파일들을 import하여 화면전환 없이 바로 가능하도록 설계하였습니다. v-model을 이용하여 이전의 댓글이 무엇이었는지 볼 수 있게 하였습니다.

게시물 리스트 검색창에서는 제목, 제목+내용, 작성자에 대한 검색이 가능하도록 하였고 이 과정에서 SQL에서 choose 와 when을 이용하여 and like의 내용을 검색 조건에 따라 분류하였습니다.

설문지 작성 창은 구글 설문지 작성을 참고하여 만들었기 때문에 객관식, 주관식, 체크박스, 드롭박스를 설문 형식으로 사용할 수 있습니다. components에 각 형식을 vue파일로 만들어 두고 componentList를 만들어 각 버튼을 누를 때 마다 components 항목이 추가되는 형식으로 설계하였습니다. 주관식을 제외한 각 항목은 ItemList에 포함되고, 모든 질문은 QuestionList에 포함됩니다. 앞서 만들어진 ItemList 또한 QuestionList에 포함된 후 Survey에 포함되게 됩니다. Spring boot에서 Map과 List를 사용하여 for문을 통해 QuestionList와 ItemList를 각각 db에 저장합니다.

설문 창은 설문지 작성과 역으로 ItemList와 QuestionList를 Survey에 담아 값을 넘겨주게 하였습니다.

설문 통계 창은 각 설문지마다 작성한 회원의 목록과 그 회원의 설문 답안을 확인할 수 있도록 하였습니다. 설문지마다 질문에 대답한 내용을 AnswerList에 담아 db에 저장하였고, 설문 작성과 마찬가지로 역순으로 AnswerList와 QuestionList를 Survey에 담아 값을 넘겨주었습니다. vs code에서 disabled를 사용하여 값을 변경할 수 없게 하였습니다. 또 vue-google-charts에서 pie chart를 이용하여 주관식을 제외한 답변들을 차트화 하여 비율과 대답횟수를 알기 쉽도록 하였습니다. 차트 또한 components에서 import 하여 화면 전환없이 볼 수 있도록 하였습니다. option에 sliceVisibilityThreshold: 0를 추가하면 갯수가 0인 항목도 차트 목록에 포함됩니다.
